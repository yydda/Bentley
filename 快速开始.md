# 快速开始指南

## 安装步骤

1. **安装依赖**
```bash
npm install
```

2. **启动开发服务器**
```bash
npm run dev
```

3. **构建生产版本**
```bash
npm run build
```

## 使用说明

### 填写日记

1. 打开应用后，默认显示今天的日期
2. 按照步骤条依次填写四个模块：
   - **时间花销**：记录各项活动的时间，点击表情符号切换评分
   - **LIFE**：记录生活习惯、内耗、反思等
   - **AM**：记录工作相关内容
   - **LOVE**：记录情感相关内容
3. 数据会自动保存到本地存储
4. 可以随时点击"导出Markdown"按钮导出日记

### 查看历史记录

- 点击顶部的日期选择器，选择要查看或编辑的日期
- 系统会自动加载该日期的数据

### 导出日记

- 点击右上角的"导出Markdown"按钮
- 系统会生成Markdown格式的文件并自动下载

## 部署到 GitHub Pages

### 方法一：使用 GitHub Actions（推荐）

1. 将代码推送到 GitHub 仓库
2. 在 GitHub 仓库设置中：
   - 进入 Settings > Pages
   - Source 选择 "GitHub Actions"
3. 每次推送到 `main` 分支时，GitHub Actions 会自动构建并部署

### 方法二：手动部署

1. 构建项目：
```bash
npm run build
```

2. 将 `dist` 目录的内容推送到 `gh-pages` 分支：
```bash
git subtree push --prefix dist origin gh-pages
```

或者使用 `gh-pages` 工具：
```bash
npm install -g gh-pages
gh-pages -d dist
```

## 注意事项

- 所有数据存储在浏览器的 LocalStorage 中
- 清除浏览器数据会导致日记丢失，建议定期导出备份
- 不同浏览器之间的数据不共享

## 常见问题

**Q: 数据丢失了怎么办？**
A: 如果之前导出过 Markdown 文件，可以重新导入。目前版本暂不支持导入功能，建议定期导出备份。

**Q: 可以在手机上使用吗？**
A: 可以，应用支持响应式设计，在手机上也能正常使用。

**Q: 如何备份数据？**
A: 可以定期导出 Markdown 文件，或者使用浏览器的数据导出功能。
